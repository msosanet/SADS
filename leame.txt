IASis - Fork paralelo del sistema legado
=======================================

Fecha de creacion: 2026-02-16
Ubicacion: d:\backsobral\Sistema\www\html\IASis

Objetivo
--------
Crear una rama paralela tecnica para comenzar separacion de logica y vistas,
sin reescribir masivamente el sistema, sin tocar el codigo original y sin
modificar conexiones a base de datos.

Que se hizo
-----------
1) Se genero una copia paralela del legado en:
   - IASis\legacy\<modulo>\...

2) Se genero estructura separada por archivo en:
   - IASis\migrado\<modulo>\logica\...
   - IASis\migrado\<modulo>\vistas\...

3) Se dejo una entrada minima para ejecutar pares logica/vista:
   - IASis\public\index.php
   - IASis\app\bootstrap.php

4) Se documento el proceso en:
   - IASis\docs\reporte_migracion.md
   - IASis\docs\reporte_migracion.csv

Reglas respetadas
-----------------
- No se editaron archivos originales fuera de IASis.
- No se cambiaron credenciales, hosts ni archivos de conexion de BD.
- Se mantuvo la mayor parte del codigo, aplicando separacion automatica conservadora.

Notas importantes
-----------------
- La separacion fue automatica por deteccion de marcadores HTML (DOCTYPE/html/body).
- En archivos donde PHP y HTML estan muy intercalados, parte de logica puede seguir
  dentro de vistas por compatibilidad (sin reescritura funcional).
- Esta base esta preparada para refactor incremental manual por modulo critico.

Uso basico
----------
Ejemplo:
  IASis\public\index.php?m=inasistencia&f=asistencia

El parametro f es la ruta relativa dentro de logica/vistas, sin extension.

Siguiente paso recomendado
--------------------------
Tomar 1 flujo critico (por ejemplo asistencia diaria) y hacer separacion manual fina
(controller/service/repository/view) sobre los archivos ya ubicados en IASis\migrado.
